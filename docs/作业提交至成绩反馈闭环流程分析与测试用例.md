# 学生作业提交至成绩反馈完整闭环流程分析与测试用例

## 一、流程分析与优化

### 1.1 当前流程梳理

#### 学生端流程
```
1. 登录系统 → 进入"我的学习" → 点击"我的课程"
2. 选择课程 → 进入课程详情页
3. 查看作业列表 → 点击"提交作业"
4. 弹出提交模态框 → 输入作业内容
5. 点击"提交" → 系统生成AI评分建议
6. 显示AI评分建议（仅查看，无确认机制）
7. ❌ 缺少：作业提交历史查看
8. ❌ 缺少：提交确认机制
```

#### 教师端流程
```
1. 登录系统 → 进入"教学管理" → 点击"我的课程"
2. 查看待评分作业列表
3. 点击作业 → 查看学生提交内容
4. 查看AI评分建议
5. 修改评分或接受AI建议
6. 点击"确认评分" → 系统更新成绩和GPA
7. 学生收到成绩通知
8. ❌ 缺少：作业状态展示（已提交/已评分）
```

#### 后端流程
```
POST /lms/submissions
  - 创建文件资产
  - 生成AI评分建议
  - 创建作业提交记录
  - 创建教师通知

GET /lms/submissions/pending/:lecturerId
  - 查询待评分作业列表
  - 包含AI评分建议

PATCH /lms/submissions/:id/grade
  - 更新作业成绩
  - 更新课程注册成绩
  - 计算并更新GPA
  - 创建学生通知
```

### 1.2 流程断点识别

| 序号 | 断点位置 | 问题描述 | 影响范围 | 优先级 |
|------|---------|---------|---------|--------|
| 1 | 学生提交作业后 | 无提交确认机制，学生无法确认提交成功 | 学生体验 | 高 |
| 2 | 学生课程详情页 | 缺少作业提交历史查看功能 | 学生体验 | 高 |
| 3 | 学生课程详情页 | 未显示作业评分状态（已提交/已评分） | 学生体验 | 高 |
| 4 | 学生收到成绩通知 | 通知点击后无跳转至成绩详情功能 | 学生体验 | 中 |
| 5 | 教师评分页面 | 缺少作业状态标签（待评分/已评分） | 教师体验 | 中 |
| 6 | AI评分建议 | 缺少接受/拒绝机制，教师需手动修改 | 教师效率 | 中 |

### 1.3 优化方案设计

#### 优化1：学生端作业提交确认机制
**实现方案**：
- 提交成功后显示确认模态框，包含：
  - 提交时间
  - 作业标题
  - AI评分建议摘要
  - "查看详情"和"返回课程"按钮
- 添加提交成功动画效果

**技术实现**：
```typescript
// LmsCourseDetailPage.tsx
const submitMutation = useMutation({
  onSuccess: (data) => {
    setShowSubmitConfirmation(true)
    setViewAI(data.data)
  }
})

// 添加确认模态框
<Modal isOpen={showSubmitConfirmation} onClose={() => setShowSubmitConfirmation(false)}>
  <div className={styles.confirmation}>
    <CheckCircle className={styles.successIcon} />
    <h3>作业提交成功</h3>
    <p>您的作业已成功提交</p>
    <div className={styles.aiSummary}>
      <h4>AI评分建议</h4>
      <p>综合评分：{aiScore}/10</p>
    </div>
    <div className={styles.actions}>
      <Button onClick={() => navigate(`/lms/submissions/${data.id}`)}>
        查看详情
      </Button>
      <Button variant="secondary" onClick={() => setShowSubmitConfirmation(false)}>
        返回课程
      </Button>
    </div>
  </div>
</Modal>
```

#### 优化2：学生端作业提交历史查看
**实现方案**：
- 在课程详情页添加"提交历史"标签页
- 显示所有作业的提交记录
- 包含提交时间、状态、成绩

**技术实现**：
```typescript
// 添加查询
const { data: submissionHistory } = useQuery({
  queryKey: ['submissions', 'history', offeringId, studentProfile?.id],
  queryFn: async () => {
    const { data } = await apiClient.get(`/lms/submissions/history/${offeringId}/${studentProfile!.id}`)
    return data.data
  },
  enabled: !!studentProfile?.id,
})

// 添加标签页切换
const [activeTab, setActiveTab] = useState<'assignments' | 'history'>('assignments')

// 显示历史记录
{activeTab === 'history' && (
  <div className={styles.historyList}>
    {submissionHistory?.map(sub => (
      <Card key={sub.id}>
        <div className={styles.historyItem}>
          <h4>{sub.assignment.title}</h4>
          <Badge status={sub.finalMarks ? 'graded' : 'pending'}>
            {sub.finalMarks ? '已评分' : '待评分'}
          </Badge>
          <p>提交时间：{formatDate(sub.submittedAt)}</p>
          {sub.finalMarks && <p>成绩：{sub.finalMarks}/{sub.assignment.maxMarks}</p>}
        </div>
      </Card>
    ))}
  </div>
)}
```

#### 优化3：课程页面作业状态展示
**实现方案**：
- 在作业列表中添加状态标签
- 已提交：显示"已提交"标签和提交时间
- 已评分：显示成绩和等级

**技术实现**：
```typescript
// 在作业卡片中添加状态显示
<Card>
  <div className={styles.assignmentCard}>
    <div className={styles.header}>
      <h4>{assignment.title}</h4>
      {submission && (
        <Badge status={submission.finalMarks ? 'graded' : 'submitted'}>
          {submission.finalMarks ? '已评分' : '已提交'}
        </Badge>
      )}
    </div>
    <p>截止日期：{formatDate(assignment.dueDate)}</p>
    {submission && (
      <div className={styles.status}>
        {submission.finalMarks ? (
          <p>成绩：{submission.finalMarks}/{assignment.maxMarks} ({submission.grade})</p>
        ) : (
          <p>提交时间：{formatDate(submission.submittedAt)}</p>
        )}
      </div>
    )}
    <Button onClick={() => setSubmitModal(assignment)}>
      {submission ? '重新提交' : '提交作业'}
    </Button>
  </div>
</Card>
```

#### 优化5：教师端作业状态展示
**实现方案**：
- 在评分页面添加状态筛选
- 显示作业状态标签（待评分/已评分）

**技术实现**：
```typescript
// 添加状态筛选
const [statusFilter, setStatusFilter] = useState<'all' | 'pending' | 'graded'>('pending')

// 筛选提交记录
const filteredSubmissions = submissions.filter(sub => {
  if (statusFilter === 'all') return true
  if (statusFilter === 'pending') return !sub.finalMarks
  if (statusFilter === 'graded') return !!sub.finalMarks
  return true
})

// 显示状态标签
<Badge status={submission.finalMarks ? 'graded' : 'pending'}>
  {submission.finalMarks ? '已评分' : '待评分'}
</Badge>
```

#### 优化6：AI评分建议接受机制
**实现方案**：
- 添加"接受AI建议"按钮
- 一键应用所有AI评分

**技术实现**：
```typescript
// LmsGradingPage.tsx
const acceptAISuggestion = () => {
  const aiScores = JSON.parse(gradingSubmission.aiRubricScores)
  setRubricScores(aiScores.map(score => ({
    ...score,
    instructor_score: score.ai_score,
    instructor_comment: score.ai_comment
  })))
  const totalScore = aiScores.reduce((sum, score) => sum + score.ai_score, 0)
  setFinalMarks(Math.round(totalScore / aiScores.length * 10))
}

// 添加接受按钮
<div className={styles.aiActions}>
  <Button onClick={acceptAISuggestion}>
    接受AI评分建议
  </Button>
</div>
```
