## 1. 邮件服务配置

### 1.1 功能概述

邮件服务是系统的基础设施，支持以下场景的邮件发送：
- 录取通知书发送
- 学生账号凭证发送
- 学费支付确认
- 退课确认通知

### 1.2 配置步骤

#### 步骤一：进入系统设置

1. 使用管理员账号登录系统
2. 点击左侧导航栏的 **「系统设置」**
3. 选择 **「邮件配置」** 选项卡

#### 步骤二：配置 SMTP 服务器

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| 启用邮件服务 | 开关控制邮件功能 | 开启 |
| SMTP服务器 | 邮件服务器地址 | smtp.gmail.com |
| SMTP端口 | 服务器端口 | 587 |
| SMTP用户名 | 发件邮箱账号 | your-email@gmail.com |
| SMTP密码 | 邮箱授权码 | •••••••• |
| 发件人名称 | 显示在邮件中的名称 | UNISSA |
| 发件人地址 | 显示的发件邮箱 | noreply@unissa.edu.bn |
| 使用SSL/TLS | 安全连接选项 | 根据服务器要求选择 |

#### 步骤三：测试邮件配置

1. 在「测试邮箱地址」输入框中填写接收测试邮件的邮箱
2. 点击 **「发送测试邮件」** 按钮
3. 查看测试结果：
   - ✅ 绿色提示：测试成功
   - ❌ 红色提示：测试失败，请检查配置

#### 步骤四：保存配置

点击 **「保存配置」** 按钮，系统将保存邮件配置并立即生效。

### 1.3 常用 SMTP 配置参考

#### Gmail 配置

```
SMTP服务器: smtp.gmail.com
端口: 587
SSL/TLS: 关闭（使用 STARTTLS）
```

> **注意**：Gmail 需要使用「应用专用密码」，而非账号登录密码。请在 Google 账户设置中开启两步验证后生成应用专用密码。

#### Outlook / Office 365 配置

```
SMTP服务器: smtp.office365.com
端口: 587
SSL/TLS: 开启
```

#### 自建邮件服务器

```
SMTP服务器: mail.unissa.edu.bn
端口: 465 或 587
SSL/TLS: 根据服务器配置
```

## 2. 录取通知书自动生成与发送

### 2.1 功能概述

当招生管理员将申请状态更改为「已录取」时，系统会自动：
1. 生成录取编号（格式：OFFER-年份-序号）
2. 生成录取通知书 PDF 文件
3. 保存 PDF 到系统文件存储
4. 发送录取通知书邮件给申请人（需邮件服务已配置）

### 2.2 操作步骤

#### 步骤一：进入入学申请管理

1. 使用招生管理员或管理员账号登录
2. 点击左侧导航栏的 **「招生管理」** → **「入学申请」**

#### 步骤二：审核申请

1. 在申请列表中找到待审核的申请
2. 点击申请记录查看详细信息
3. 确认申请人资料完整且符合录取条件

#### 步骤三：执行录取操作

1. 点击 **「录取」** 按钮
2. 可选：填写备注说明
3. 确认操作

#### 步骤四：查看结果

录取成功后：
- 系统自动生成录取通知书 PDF
- 申请人状态变更为「已录取」
- 如果邮件服务已配置，申请人将收到录取通知书邮件

### 2.3 录取通知书内容

录取通知书 PDF 包含以下信息：

| 内容项 | 说明 |
|--------|------|
| 学校标识 | UNISSA 校名和地址 |
| 录取编号 | 唯一标识码，格式：OFFER-2026-XXXX |
| 录取日期 | 录取决定日期 |
| 申请人信息 | 姓名、护照/身份证号 |
| 课程信息 | 课程名称、学习模式、学制 |
| 入学信息 | 入学学期、开学日期 |
| 确认截止 | 接受录取的截止日期 |
| 联系方式 | 招生办公室联系方式 |

### 2.4 邮件内容示例

```
主题：Admission Offer - OFFER-2026-1234 | UNISSA

正文：
Congratulations, [申请人姓名]!

We are pleased to inform you that your application to UNISSA has been successful.
You have been offered admission to the [课程名称] programme.

Please find your official offer letter attached to this email.

Offer Reference: OFFER-2026-1234

Next Steps:
1. Review your offer letter
2. Log in to the UNISSA Student Portal to accept your offer
3. Complete your course registration
```

## 3. 退课账单自动调整

### 3.1 功能概述

当学生退选课程时，系统会自动：
1. 计算应退金额（学分 × 单价）
2. 更新学费账单金额
3. 调整账单状态
4. 记录调整历史
5. 发送退课确认邮件（需邮件服务已配置）

### 3.2 退课操作步骤

#### 学生端操作

1. 登录学生账号
2. 进入 **「我的校园」** → **「选课」**
3. 找到已选课程列表
4. 点击目标课程的 **「退选」** 按钮
5. 确认退选操作

#### 管理员端操作

1. 登录管理员账号
2. 进入学生管理或选课管理
3. 找到目标学生的选课记录
4. 执行退选操作

### 3.3 退款计算规则

```
退款金额 = 课程学分 × 学分单价

学分单价：
- 文莱本地学生：使用课程设定的本地学生单价
- 国际学生：使用课程设定的国际学生单价
```

### 3.4 账单状态调整

| 原状态 | 退款后状态 | 条件 |
|--------|------------|------|
| unpaid | unpaid | 仍有欠款 |
| unpaid | paid | 欠款归零 |
| partial | partial | 仍有欠款 |
| partial | paid | 欠款归零 |
| paid | paid | 已付款部分转为预付款/退款 |

### 3.5 调整记录

每次退课调整都会记录以下信息：
- 调整类型（refund）
- 调整金额
- 调整原因（退课课程信息）
- 调整前后金额
- 调整前后状态
- 操作人员
- 操作时间

### 3.6 退课确认邮件示例

```
主题：Course Drop Confirmation - [课程代码]

正文：
Dear [学生姓名],

You have successfully dropped the following course:

Course: [课程代码] - [课程名称]
Credit Hours: [学分]
Refund Amount: BND [金额]

Your invoice has been updated accordingly.

UNISSA - Universiti Islam Sultan Sharif Ali
Finance Department
```

## 4. 邮件日志查看

### 4.1 功能概述

管理员可以查看系统发送的所有邮件记录，包括发送状态、时间、收件人等信息。

### 4.2 查看步骤

1. 登录管理员账号
2. 进入 **「系统设置」** → **「邮件配置」**
3. 滚动到页面底部查看邮件日志区域

### 4.3 日志字段说明

| 字段 | 说明 |
|------|------|
| 收件人 | 邮件接收者邮箱地址 |
| 主题 | 邮件主题 |
| 状态 | sent（已发送）/ failed（发送失败） |
| 发送时间 | 邮件发送成功的时间 |
| 重试次数 | 发送失败后的重试次数 |
| 错误信息 | 发送失败时的错误描述 |

### 4.4 状态筛选

可通过状态筛选查看：
- 全部邮件
- 已发送邮件
- 发送失败邮件

### 4.5 API 接口

管理员也可通过 API 获取邮件日志：

```http
GET /api/v1/admin/email-logs?page=1&limit=20&status=sent
```

响应示例：

```json
{
  "success": true,
  "data": [
    {
      "id": "clx123456",
      "to": "student@example.com",
      "subject": "Admission Offer - OFFER-2026-1234",
      "status": "sent",
      "sentAt": "2026-04-04T10:30:00Z",
      "retries": 0

}
```

## 5. 常见问题

### 5.1 邮件发送失败

**问题**：测试邮件发送失败

**排查步骤**：

1. **检查 SMTP 配置**
   - 确认服务器地址和端口正确
   - 确认用户名和密码正确
   - 检查 SSL/TLS 设置是否与服务器要求一致

2. **检查网络连接**
   - 确认服务器可以访问外网
   - 检查防火墙是否阻止了 SMTP 端口

3. **检查邮箱授权**
   - Gmail 等邮箱需要使用应用专用密码
   - 确认邮箱已开启 SMTP 发送权限

4. **查看错误日志**
   - 在邮件日志中查看具体错误信息
   - 根据错误信息进行针对性处理

### 5.2 录取通知书未生成

**问题**：录取操作后未生成 PDF

**排查步骤**：

1. 检查申请人信息是否完整
2. 检查课程和入学学期信息是否存在
3. 查看后端日志是否有错误信息
4. 确认 uploads 目录有写入权限

### 5.3 退课金额计算不正确

**问题**：退课退款金额与预期不符

**排查步骤**：

1. 检查学生的国籍设置（本地/国际学生）
2. 检查课程的学分设置
3. 检查课程的学分单价设置
4. 确认账单是否已存在

### 5.4 邮件服务未启用

**问题**：配置保存后邮件服务仍显示未启用

**排查步骤**：

1. 确认「启用邮件服务」开关已开启
2. 点击保存配置按钮
3. 刷新页面重新检查状态
4. 调用 `/api/v1/admin/email-status` 检查服务状态

#### EmailLog 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 主键 |
| to | String | 收件人 |
| subject | String | 邮件主题 |
| body | String | 邮件内容 |
| status | String | 状态（pending/sent/failed） |
| errorMessage | String? | 错误信息 |
| messageId | String? | 邮件服务器返回的消息ID |
| sentAt | DateTime? | 发送时间 |
| retries | Int | 重试次数 |
| createdAt | DateTime | 创建时间 |

#### InvoiceAdjustment 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 主键 |
| invoiceId | String | 关联账单ID |
| adjustmentType | String | 调整类型（refund/charge） |
| amount | Float | 调整金额 |
| reason | String | 调整原因 |
| previousAmount | Float | 调整前金额 |
| newAmount | Float | 调整后金额 |
| previousStatus | String | 调整前状态 |
| newStatus | String | 调整后状态 |

| createdAt | DateTime | 创建时间 |
