## 2. 学生上课模块测试用例

### 2.1 课程列表查看 (S-001)

#### 2.1.1 正常操作场景

| 用例编号 | TC-S001-01 |
|----------|------------|
| 用例名称 | 学生查看已注册课程列表 |
| 前置条件 | 1. 学生已登录系统<br>2. 学生已注册至少一门课程 |
| 测试步骤 | 1. 学生登录系统<br>2. 点击「我的课程」或「LMS」菜单<br>3. 系统显示课程列表页面 |
| 预期结果 | 1. 显示所有已注册课程<br>2. 每门课程显示课程代码、名称、学分、教师、时间、地点<br>3. 显示待完成作业数量<br>4. 显示已获得成绩（如有） |
| 优先级 | P0 |

| 用例编号 | TC-S001-02 |
|----------|------------|
| 用例名称 | 学生切换卡片/日历视图 |
| 前置条件 | 1. 学生已登录系统<br>2. 学生已注册课程 |
| 测试步骤 | 1. 进入课程列表页面<br>2. 点击日历视图按钮<br>3. 观察课程以日历形式展示<br>4. 点击卡片视图按钮 |
| 预期结果 | 1. 日历视图按周显示课程安排<br>2. 卡片视图显示课程卡片列表<br>3. 切换过程流畅无卡顿 |
| 优先级 | P1 |

#### 2.1.2 边界条件场景

| 用例编号 | TC-S001-03 |
|----------|------------|
| 用例名称 | 学生未注册任何课程 |
| 前置条件 | 1. 学生已登录系统<br>2. 学生未注册任何课程 |
| 测试步骤 | 1. 进入课程列表页面 |
| 预期结果 | 1. 显示空状态提示「暂无课程」<br>2. 提供「去选课」按钮引导用户 |
| 优先级 | P2 |

| 用例编号 | TC-S001-04 |
|----------|------------|
| 用例名称 | 课程列表包含大量课程 |
| 前置条件 | 1. 学生已登录系统<br>2. 学生注册了10门以上课程 |
| 测试步骤 | 1. 进入课程列表页面<br>2. 观察页面加载和显示 |
| 预期结果 | 1. 页面正常加载<br>2. 所有课程正确显示<br>3. 支持滚动浏览<br>4. 无性能问题 |
| 优先级 | P2 |

#### 2.1.3 异常处理场景

| 用例编号 | TC-S001-05 |
|----------|------------|
| 用例名称 | 网络异常时加载课程列表 |
| 前置条件 | 1. 学生已登录系统 |
| 测试步骤 | 1. 断开网络连接<br>2. 尝试进入课程列表页面 |
| 预期结果 | 1. 显示网络错误提示<br>2. 提供重试按钮<br>3. 不显示空白页面 |
| 优先级 | P1 |

| 用例编号 | TC-S001-06 |
|----------|------------|
| 用例名称 | 会话过期后访问课程列表 |
| 前置条件 | 1. 学生登录后长时间未操作<br>2. 会话已过期 |
| 测试步骤 | 1. 会话过期后尝试访问课程列表 |
| 预期结果 | 1. 提示会话已过期<br>2. 自动跳转到登录页面<br>3. 登录后返回原页面 |
| 优先级 | P1 |

| 测试步骤 | 1. 直接通过 URL 访问未注册课程的详情页 |
| 预期结果 | 1. 显示「您未注册此课程」提示<br>2. 不显示课程内容<br>3. 提供返回课程列表的链接 |
| 优先级 | P0 |

| 用例编号 | TC-S002-04 |
|----------|------------|
| 用例名称 | 课程无任何材料或作业 |
| 前置条件 | 1. 学生已注册该课程<br>2. 课程无材料、无作业 |
| 测试步骤 | 1. 进入课程详情页<br>2. 切换到各标签页 |
| 预期结果 | 1. 各标签页显示空状态提示<br>2. 提示内容友好明确 |
| 优先级 | P2 |

#### 2.2.3 权限控制场景

| 用例编号 | TC-S002-05 |
|----------|------------|
| 用例名称 | 学生尝试访问其他学生的课程 |

| 测试步骤 | 1. 学生 A 通过 URL 尝试访问课程 X 的详情页 |
| 预期结果 | 1. 返回 403 权限错误<br>2. 显示「您没有权限访问此课程」 |
| 优先级 | P0 |

| 预期结果 | 1. 显示「暂无课程材料」提示 |
| 优先级 | P2 |

| 用例编号 | TC-S003-05 |

| 用例名称 | 材料文件已被删除 |

### 2.4 作业提交 (S-004)

#### 2.4.1 正常操作场景

| 用例编号 | TC-S004-01 |
|----------|------------|
| 用例名称 | 学生提交文本内容作业 |
| 前置条件 | 1. 学生已进入课程详情页<br>2. 存在未截止的作业<br>3. 学生未提交过该作业 |
| 测试步骤 | 1. 进入「作业」标签页<br>2. 点击某作业的「提交」按钮<br>3. 在文本框输入作业内容<br>4. 点击「提交」按钮 |
| 预期结果 | 1. 显示 AI 评分加载状态<br>2. 提交成功后显示确认弹窗<br>3. 作业列表更新为「已提交」状态<br>4. 显示 AI 评分结果 |
| 优先级 | P0 |

| 用例编号 | TC-S004-02 |
|----------|------------|
| 用例名称 | 学生提交带附件的作业 |
| 前置条件 | 1. 存在未截止的作业 |
| 测试步骤 | 1. 点击「提交」按钮<br>2. 输入文本内容<br>3. 点击上传附件按钮<br>4. 选择图片文件（JPG/PNG）<br>5. 点击「提交」 |
| 预期结果 | 1. 附件上传成功<br>2. 显示附件预览<br>3. 提交成功后显示附件信息 |
| 优先级 | P0 |

| 用例编号 | TC-S004-03 |
|----------|------------|
| 用例名称 | 查看已提交作业的 AI 评分 |
| 前置条件 | 1. 学生已提交作业<br>2. AI 已完成评分 |
| 测试步骤 | 1. 在作业列表找到已提交作业<br>2. 点击「AI 评分」按钮 |
| 预期结果 | 1. 弹窗显示 AI 评分详情<br>2. 显示各评分维度得分和评语<br>3. 显示改进建议 |
| 优先级 | P1 |

#### 2.4.2 边界条件场景

|----------|------------|
| 用例名称 | 提交截止日期已过的作业 |

| 预期结果 | 1. 作业显示「已截止」标签<br>2. 不显示「提交」按钮<br>3. 显示红色截止日期提示 |

## 6. 改进建议

### 6.1 测试用例改进建议

#### 6.1.1 结构化改进

```
建议采用以下测试用例模板：

| 字段 | 说明 |
|------|------|
| 用例编号 | 唯一标识，格式：TC-[模块]-[功能]-[序号] |

| 前置条件 | 详细列出所有依赖条件，包括数据准备 |
| 测试数据 | 明确测试所需的具体数据值 |
| 测试步骤 | 分步骤详细描述操作，包含等待和验证 |
| 预期结果 | 具体描述UI、API、数据库三方面的预期 |
| 后置处理 | 测试后的数据清理操作 |
| 优先级 | P0-P3，基于业务风险评估 |
| 自动化标识 | 是否适合自动化测试 |
```

3. **数据覆盖率**：补充边界值和特殊数据测试
4. **角色覆盖率**：确保所有用户角色有对应测试

### 6.2 测试数据管理建议

```
建议建立测试数据管理规范：

1. 测试数据隔离
   - 使用独立测试数据库
   - 每次测试运行前重置数据

   - 创建测试数据生成工具
   - 支持快速生成各类测试数据

describe('validateSubmissionFiles', () => {
  it('should accept valid image files', () => {
    const files = [new File([''], 'test.jpg', { type: 'image/jpeg' })]
    const result = validateFiles(files)
    expect(result.valid).toBe(true)
  })

  it('should reject files larger than 10MB', () => {
    const largeFile = new File(['x'.repeat(11 * 1024 * 1024)], 'large.jpg', { type: 'image/jpeg' })
    const result = validateFiles([largeFile])
    expect(result.valid).toBe(false)
    expect(result.errors).toContain('File size exceeds limit')
  })

  it('should reject more than 5 files', () => {
    const files = Array(6).fill(null).map((_, i) => 
      new File([''], `test${i}.jpg`, { type: 'image/jpeg' })
    )
    const result = validateFiles(files)
    expect(result.valid).toBe(false)
  })
})
```
