# 课程管理模块使用手册

> **适用系统**：UNISSA POC v2
> **模块路径**：侧边栏 → 管理 → 课程管理（`/admin/courses`）
> **支持角色**：管理员（Admin）

---

## 一、演示账号

| 用户名 | 显示名 | 角色 | 密码 |
|--------|--------|------|------|
| `admin` | System Admin | admin | `Demo@2026` |

---

## 二、功能概览

课程管理页面允许管理员对课程目录进行完整的 CRUD 操作：

| 功能 | 说明 |
|------|------|
| 查看课程列表 | 以表格形式展示所有课程，支持按代码/名称搜索 |
| 新增课程 | 填写表单新增课程，自动校验课程代码是否重复 |
| 编辑课程 | 修改课程名称、院系、学分、年级、人数上限等信息 |
| 删除课程 | 删除无排课的课程，有排课时系统自动拦截 |

---

## 三、操作流程

### 第一步：登录并进入模块

1. 访问系统首页，输入用户名 `admin`，密码 `Demo@2026`
2. 登录后点击左侧导航栏 **"管理"** 分组，选择 **"课程管理"**

---

### 第二步：查看课程列表

页面以表格形式展示系统内所有课程，列信息如下：

| 列名 | 说明 |
|------|------|
| **课程代码** | 唯一标识符，如 `IFN101` |
| **课程名称** | 完整课程名称 |
| **院系** | 所属院系名称 |
| **学分** | 学时数（Credit Hours） |
| **年级** | 课程所在年级层次（Level 1–4） |
| **最大人数** | 每个排课的座位上限 |
| **排课数** | 当前学期该课程的排课数量 |
| **国际生** | 是否对国际学生开放 |
| **操作** | 编辑 / 删除按钮 |

**搜索过滤**：在页面左上角搜索框中输入课程代码或课程名称关键字，表格实时过滤。

---

### 第三步：新增课程

1. 点击页面右上角 **"新增课程"** 按钮
2. 在弹窗表单中填写以下字段：

   | 字段 | 必填 | 说明 |
   |------|------|------|
   | **课程代码** | ✅ | 大写英数字，如 `IFN301`（创建后不可修改） |
   | **院系** | ✅ | 从下拉列表选择所属院系 |
   | **课程名称** | ✅ | 课程完整名称 |
   | **学分** | — | 默认 3，可选 1–6 |
   | **年级** | — | 默认 Level 1，可选 Level 1–4 |
   | **最大人数** | — | 默认 40，可填 1–500 |
   | **对国际生开放** | — | 默认开启 |

3. 点击 **"新增课程"** 确认
4. 成功后弹出绿色提示"课程创建成功"，表格自动刷新

> **注意**：若课程代码已存在，系统会提示冲突错误，需更换代码后重新提交。

---

### 第四步：编辑课程

1. 在表格中找到目标课程，点击操作列的 ✏️ **编辑按钮**
2. 弹窗自动填入当前数据（课程代码不可修改）
3. 修改所需字段后点击 **"保存"**
4. 成功后提示"课程更新成功"

---

### 第五步：删除课程

1. 在操作列点击 🗑️ **删除按钮**
2. 弹出确认提示，显示课程代码及当前排课数
3. 若该课程**有排课**，确认弹窗会显示警告提示，提交后后端也会拦截并返回错误
4. 若该课程**无排课**，点击红色 **"删除"** 按钮确认执行
5. 成功后提示"课程删除成功"，表格自动更新

> **安全机制**：有排课（CourseOffering）的课程无法删除，需先在数据库中清除关联排课记录。

## 五、常见问题

**Q：课程代码已经存在，提交失败怎么办？**
A：系统对课程代码做唯一性校验。请换用不同的代码（如 `IFN302`）重新提交。

**Q：为什么有些课程不能删除？**
A：该课程存在关联的 `CourseOffering`（排课记录）。需先删除相应排课后才能删除课程。当前 POC 版本排课删除需通过数据库操作，暂无 UI 入口。

**Q：院系下拉列表为空怎么办？**
A：请确认已运行 `pnpm prisma db seed` 完成数据初始化，院系数据由种子脚本写入。

**Q：新增课程后学生可以立即选课吗？**
A：不能。新建课程后需由系统管理员在数据库中创建对应的 `CourseOffering`（排课记录，包含学期、讲师、时间、教室等信息），学生才能在选课页看到并注册该课程。

**Q：编辑时课程代码为何置灰无法修改？**
A：课程代码是数据库主键的业务唯一键，修改会影响所有关联数据。如需更改代码，请先删除旧课程再新建。

## 六、技术说明（供开发参考）

| 项目 | 说明 |
|------|------|
| 数据模型 | `Course`（课程）、`Department`（院系）、`CourseOffering`（排课） |
| 课程代码 | `@@unique` 约束，提交时自动转大写（`toUpperCase()`） |
| 删除保护 | 后端先 `count` 关联 `CourseOffering`，大于 0 时返回 409 |
| 权限控制 | 全部路由使用 `requireRole('admin')` 中间件 |
| 院系数据 | 由 `GET /api/v1/admin/departments` 返回，前端缓存至 React Query |

### API 端点汇总

| 方法 | 路径 | 权限 | 功能 |
|------|------|------|------|
| GET | `/api/v1/admin/departments` | admin | 获取院系列表（表单下拉数据源） |
| GET | `/api/v1/admin/courses` | admin | 获取所有课程（含排课数统计） |
| POST | `/api/v1/admin/courses` | admin | 新增课程 |
| PUT | `/api/v1/admin/courses/:id` | admin | 更新课程信息 |
| DELETE | `/api/v1/admin/courses/:id` | admin | 删除课程（有排课时返回 409） |
