## 一、功能概述

### 1.1 背景与目标

基于演示路径完整规划文档的分析，识别出以下四个最高优先级的功能增强需求：

| 优先级 | 功能 | 影响范围 | 业务价值 |
|--------|------|----------|----------|
| P0 | 邮件服务集成 | 全系统 | 实现真实邮件发送，提升用户体验 |
| P0 | 录取通知书自动生成 | S1→S2 | 自动化录取流程，减少人工操作 |
| P1 | 用户账号自动创建 | S2 | 简化入职流程，提升效率 |
| P1 | 退课后账单自动调整 | S3→S4 | 保证财务数据准确性 |

### 1.2 功能依赖关系

```
邮件服务集成 (P0)
    ├── 录取通知书自动生成 (P0) ──依赖──▶ 邮件发送
    └── 用户账号自动创建 (P1) ──依赖──▶ 邮件发送账号凭据

退课后账单自动调整 (P1) ──独立──▶ 无外部依赖
```

#### 账号创建流程

```

    │
    ▼

    │

    │
    └── 不存在 ──▶ 创建新账号
                        │
                        ▼
                   生成用户名和密码
                        │
                        ▼

                        │
                        ▼
                   发送账号凭据邮件

**1. 邮件配置管理 (`backend/src/services/emailService.ts`)**

```typescript
import nodemailer from 'nodemailer'
import { prisma } from '@/lib/prisma'

interface EmailConfig {
  host: string
  port: number
  secure: boolean
  user: string
  pass: string
  fromName: string
  fromAddress: string
}

class EmailService {
  private transporter: nodemailer.Transporter | null = null
  private config: EmailConfig | null = null

  async initialize(): Promise<void> {
    const configs = await prisma.systemConfig.findMany({
      where: { key: { startsWith: 'email_' } }
    })
    
    this.config = {
      host: configs.find(c => c.key === 'email_host')?.value || 'smtp.gmail.com',
      port: parseInt(configs.find(c => c.key === 'email_port')?.value || '587'),
      secure: configs.find(c => c.key === 'email_secure')?.value === 'true',
      user: configs.find(c => c.key === 'email_user')?.value || '',
      pass: configs.find(c => c.key === 'email_pass')?.value || '',
      fromName: configs.find(c => c.key === 'email_from_name')?.value || 'UNISSA',
      fromAddress: configs.find(c => c.key === 'email_from_address')?.value || 'noreply@unissa.edu.bn',
    }

    if (this.config.user && this.config.pass) {
      this.transporter = nodemailer.createTransport({
        host: this.config.host,
        port: this.config.port,
        secure: this.config.secure,
        auth: {
          user: this.config.user,
          pass: this.config.pass,
        },
      })
    }
  }

  async sendEmail(options: {
    to: string | string[]
    subject: string
    body: string
    attachments?: Array<{ filename: string; content: Buffer; contentType: string }>
  }): Promise<{ success: boolean; messageId?: string; error?: string }> {
    if (!this.transporter) {
      return { success: false, error: 'Email service not configured' }
    }

    try {
      const result = await this.transporter.sendMail({
        from: `"${this.config!.fromName}" <${this.config!.fromAddress}>`,
        to: Array.isArray(options.to) ? options.to.join(',') : options.to,
        subject: options.subject,
        html: options.body,
        attachments: options.attachments?.map(a => ({
          filename: a.filename,
          content: a.content,
          contentType: a.contentType,
        })),
      })
      return { success: true, messageId: result.messageId }
    } catch (error: any) {
      return { success: false, error: error.message }
    }
  }

  async sendTestEmail(to: string): Promise<boolean> {
    const result = await this.sendEmail({
      to,
      subject: 'UNISSA Email Service Test',
      body: '<p>This is a test email from UNISSA system.</p>',
    })
    return result.success
  }
}

export const emailService = new EmailService()
```

**2. 录取通知书生成服务 (`backend/src/services/offerLetterService.ts`)**

```typescript
import PDFDocument from 'pdfkit'
import { Applicant, Programme, Intake } from '@prisma/client'

interface OfferLetterData {
  applicant: Applicant
  programme: Programme
  intake: Intake
  offerRef: string
  offerDate: Date
  confirmDeadline: Date
}

export async function generateOfferLetterPDF(data: OfferLetterData): Promise<Buffer> {
  return new Promise((resolve, reject) => {
    const doc = new PDFDocument({
      size: 'A4',
      margins: { top: 60, bottom: 60, left: 60, right: 60 },
    })

    const chunks: Buffer[] = []
    doc.on('data', (chunk: Buffer) => chunks.push(chunk))
    doc.on('end', () => resolve(Buffer.concat(chunks)))
    doc.on('error', reject)

    const { applicant, programme, offerRef, offerDate, confirmDeadline } = data

    doc.fontSize(18).font('Helvetica-Bold')
      .text('UNISSA - Universiti Islam Sultan Sharif Ali', { align: 'center' })
    doc.fontSize(10).font('Helvetica')
      .text('Bandar Seri Begawan, Brunei Darussalam', { align: 'center' })
    doc.moveDown(2)

    doc.fontSize(16).font('Helvetica-Bold')
      .text('OFFER OF ADMISSION', { align: 'center' })
    doc.moveDown()

    doc.fontSize(10).font('Helvetica')
    doc.text(`Offer Reference: ${offerRef}`, { align: 'right' })
    doc.text(`Date: ${offerDate.toLocaleDateString('en-GB', { day: 'numeric', month: 'long', year: 'numeric' })}`, { align: 'right' })
    doc.moveDown()

    doc.font('Helvetica')
      .text(`Dear ${applicant.fullName},`)
    doc.moveDown()

    doc.text(`We are pleased to offer you admission to the ${programme.name} programme for the ${data.intake.name} intake, commencing ${data.intake.intakeStart.toLocaleDateString('en-GB', { month: 'long', year: 'numeric' })}.`)
    doc.moveDown()

    doc.font('Helvetica-Bold').text('Programme Details:')
    doc.font('Helvetica')
    doc.text(`  Programme: ${programme.name}`)
    doc.text(`  Programme Code: ${programme.code}`)
    doc.text(`  Mode of Study: ${applicant.modeOfStudy?.replace('_', ' ').toUpperCase()}`)
    doc.text(`  Duration: ${programme.durationYears || 4} years`)
    doc.moveDown()

    doc.font('Helvetica-Bold').text('Important Dates:')
    doc.font('Helvetica')
    doc.text(`  Semester Start: ${data.intake.intakeStart.toLocaleDateString('en-GB')}`)
    doc.text(`  Acceptance Deadline: ${confirmDeadline.toLocaleDateString('en-GB')}`)
    doc.moveDown()

    doc.text(`Please confirm your acceptance by logging into the UNISSA Student Portal before the deadline above.`)
    doc.moveDown()

    doc.text('Congratulations on your admission to UNISSA!')
    doc.moveDown(2)

    doc.font('Helvetica-Bold').text('Sincerely,')

    doc.text('_________________________')

    doc.fontSize(8).font('Helvetica').fillColor('#666666')
    doc.text('This is a computer-generated document. For inquiries, please contact admissions@unissa.edu.bn', { align: 'center' })

    doc.end()
  })

## 九、风险管理

### 9.1 风险识别

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| SMTP 服务不稳定 | 中 | 高 | 实现重试机制，准备备用 SMTP |
| PDF 生成性能问题 | 低 | 中 | 缓存生成的 PDF，异步处理 |
| 邮件被标记为垃圾邮件 | 中 | 高 | 配置 SPF/DKIM，使用专业邮件服务 |
| 账单计算错误 | 低 | 高 | 充分的单元测试，财务审核 |

### 9.2 回滚计划

```
如果功能出现严重问题:

1. 立即禁用邮件发送功能
   - 设置 SystemConfig.email_enabled = false
   - 系统降级为仅记录日志

2. 回滚数据库变更
   - 执行回滚迁移脚本
   - 恢复上一版本数据

3. 恢复上一版本代码
   - git revert 到稳定版本
   - 重新部署服务

4. 通知相关方
   - 发送系统维护通知
   - 更新状态页面
```
