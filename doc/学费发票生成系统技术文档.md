# 学费发票生成系统技术文档

## 概述

本文档描述了UNISSA学生管理系统中学费发票生成和下载功能的实现细节。该功能允许学生用户在完成学费支付后，生成并下载PDF格式的学费发票。

## 功能特性

### 1. 发票生成功能
- 根据学生基本信息自动填充发票模板
- 包含完整的缴费信息（学费、图书馆费、宿舍押金等）
- 自动计算奖学金抵扣金额
- 显示付款历史和交易记录

### 2. 发票预览功能
- 提供美观的发票预览界面
- 显示发票抬头、学生信息、缴费明细
- 实时显示付款状态（已付/未付/逾期）
- 支持查看付款历史记录

### 3. PDF下载功能
- 生成符合标准发票格式的PDF文档
- 支持直接下载到本地
- 文件名格式：`invoice-{发票编号}.pdf`

#### 下载功能实现

```typescript
const downloadInvoice = async (invoiceId: string, invoiceNo: string) => {
  const response = await apiClient.get(`/finance/invoices/${invoiceId}/pdf`, {
    responseType: 'blob',
  })
  const url = window.URL.createObjectURL(new Blob([response.data], { type: 'application/pdf' }))
  const link = document.createElement('a')
  link.href = url
  link.setAttribute('download', `invoice-${invoiceNo}.pdf`)
  document.body.appendChild(link)
  link.click()
  link.remove()
  window.URL.revokeObjectURL(url)
}
```

## 使用流程

### 学生用户操作步骤

1. **登录系统**
   - 使用学生账号登录
   - 进入个人中心

2. **访问缴费记录**
   - 导航至"缴费记录"页面
   - 查看所有发票列表

3. **预览发票**
   - 点击发票卡片上的"预览"按钮
   - 在模态框中查看完整发票内容

4. **下载发票**
   - 在预览界面点击"下载PDF"按钮
   - 或在发票卡片直接点击"下载"按钮
   - PDF文件将自动下载到本地

## 数据模型

### FeeInvoice 模型

```prisma
model FeeInvoice {
  id                   String   @id @default(cuid())
  invoiceNo            String   @unique
  studentId            String
  student              Student  @relation(fields: [studentId], references: [id])
  semesterId           String
  semester             Semester @relation(fields: [semesterId], references: [id])
  tuitionFee           Float
  libraryFee           Float    @default(50.00)
  hostelDeposit        Float    @default(0.00)
  scholarshipDeduction Float    @default(0.00)
  totalAmount          Float
  amountPaid           Float    @default(0.00)
  outstandingBalance   Float
  dueDate              DateTime
  status               String   @default("unpaid")
  payments             Payment[]
}
```

### Payment 模型

```prisma
model Payment {
  id             String     @id @default(cuid())
  transactionRef String     @unique
  invoiceId      String
  invoice        FeeInvoice @relation(fields: [invoiceId], references: [id])
  amount         Float
  method         String
  status         String     @default("pending")
  paidAt         DateTime?
  cardLast4      String?
  cardBrand      String?
  bankName       String?
}
```

## 安全考虑

1. **身份验证**: 所有发票相关接口需要JWT认证
2. **权限控制**: 学生只能访问自己的发票
3. **数据验证**: 验证发票ID的有效性
4. **文件安全**: 生成的PDF不包含敏感信息

## 性能优化

1. **PDF生成**: 使用流式生成，减少内存占用
2. **缓存策略**: 可考虑缓存已生成的PDF
3. **异步处理**: 大文件下载使用异步处理
