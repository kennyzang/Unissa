# 部署排障：数据库初始化验证

> 适用场景：部署到阿里云后登录返回 401

---

## 排查流程

### 第一步：确认数据库文件存在

SSH 进服务器后执行：

```bash
# 找到数据库文件
find /path/to/project -name "*.db" 2>/dev/null

# 查看文件大小（若为 0 或不存在则说明未初始化）
ls -lh /path/to/backend/prisma/dev.db
```

### 第二步：查询 users 表是否有数据

```bash
sqlite3 /path/to/backend/prisma/dev.db "SELECT username, role, isActive FROM users;"
```

- 有记录 → 数据库正常，跳到第四步
- 空结果 → Seed 未执行，进行第三步

### 第三步：重新初始化数据库

```bash
cd /path/to/backend

# 1. 建表（执行 migration）
npx prisma migrate deploy

# 2. 写入演示数据（执行 seed）
pnpm seed
# 或
npx ts-node prisma/seed.ts
```

成功输出示例：
```
🌱 Seeding UNISSA demo data...
✅ Done.
```

### 第四步：直接测试登录接口

绕过前端，用 curl 验证后端接口：

```bash
curl -X POST http://your-server-ip:3000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"Demo@2026"}'
```

| 返回结果 | 说明 | 处理方式 |
|----------|------|----------|
| `{"success":true,...}` | 后端正常 | 检查前端 API 地址配置 |
| `401 Invalid credentials` | 无用户数据 | 重跑 seed |
| 连接超时 / ECONNREFUSED | 服务未启动或端口未开放 | 检查进程和安全组 |

## 常见问题

### 问题：migrate deploy 报错找不到数据库

检查 `.env` 中 `DATABASE_URL` 路径是否正确：

```bash
cat /path/to/backend/.env | grep DATABASE_URL
# 示例：DATABASE_URL="file:./prisma/dev.db"
```

路径为相对路径时，需确保在 `backend/` 目录下执行命令。

### 问题：seed 执行成功但登录仍 401

可能是密码大小写或空格问题，直接用 sqlite3 确认密码 hash 写入：

```bash
sqlite3 /path/to/backend/prisma/dev.db \
  "SELECT username, passwordHash FROM users WHERE username='admin';"
```

hash 应以 `$2b$` 开头（bcrypt 格式）。

### 问题：阿里云安全组未放行端口

确认后端端口（默认 3000）已在阿里云控制台 **安全组入方向规则** 中开放。
