## 五、测试基础设施

### 5.1 技术栈

| 工具 | 版本 | 用途 |
|------|------|------|
| Vitest | ^2.1.1 | 测试运行器 |
| @testing-library/react | ^16.0.1 | React组件测试 |
| @testing-library/jest-dom | ^6.5.0 | DOM断言扩展 |
| @testing-library/user-event | ^14.5.2 | 用户交互模拟 |
| @vitest/coverage-v8 | ^2.1.1 | 代码覆盖率 |
| jsdom | ^25.0.1 | 浏览器环境模拟 |

### 5.2 测试配置

- **环境**: jsdom
- **全局API**: 启用 (describe/it/expect/vi)
- **自动清理**: 每个测试后自动 cleanup
- **路径别名**: `@` → `./src`
- **覆盖率提供者**: v8
- **覆盖率排除**: node_modules, test目录, 类型声明, 配置文件, 入口文件

### 5.3 Mock策略

- **API客户端**: 通过 `vi.mock('@/lib/apiClient')` 模拟所有API请求
- **浏览器API**: mock matchMedia, ResizeObserver, IntersectionObserver, localStorage, sessionStorage
- **国际化**: mock i18next 的 t 函数，返回键名作为默认翻译
- **React Query**: 每个测试创建独立 QueryClient，禁用重试和缓存
