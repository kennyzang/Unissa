## 二、讲师操作流程

### 第一步：登录系统

1. 访问系统首页，输入用户名 `drsiti`，密码 `Demo@2026`
2. 登录后点击左侧导航栏 **"考勤管理"**

### 第二步：查看课程列表

登录后页面显示该讲师名下的所有课程，每条课程卡片显示：
- 课程代码与名称（如 `IFN101 – Introduction to Informatics`）
- 上课时间、教室
- 已选课学生数 / 历史会话次数

### 第三步：开启签到会话

1. 找到目标课程，点击右侧 **"开启签到"** 按钮
2. 系统自动生成一个有效期 **30 分钟** 的签到会话
3. 页面顶部弹出 **活动会话面板**，包含：

   | 内容 | 说明 |
   |------|------|
   | **二维码** | 学生用手机扫描即可签到 |
   | **会话令牌** | 可点击「复制」按钮，粘贴发送给学生 |
   | **实时人数** | 每 5 秒自动刷新已签到人数 |
   | **过期时间** | 显示二维码何时失效 |

### 第四步：分享令牌（无扫码场景）

若学生无法扫码，讲师可通过以下方式分享令牌：
1. 点击会话面板中的 **"复制"** 按钮，令牌自动复制到剪贴板
2. 将令牌粘贴到群聊、公告或投影仪展示
3. 学生在学生端 **"令牌手动签到"** 输入框粘贴后点击「签到」

> **令牌格式**：为 JWT 字符串（以 `eyJ` 开头），有效期与二维码相同（30 分钟）

### 第五步：结束会话

- 点击面板右上角 **"结束会话"** 按钮手动关闭
- 或令牌到期后会话自动失效（已签到记录不受影响）

### 查看历史记录

点击课程卡片右侧的 **展开箭头（↓）**，可查看该课程历史签到会话列表，包括：
- 会话日期与时间段
- 已签到人数
- 会话状态（进行中 / 已关闭 / 已过期）

### 方式一：扫描二维码（推荐）

1. 讲师投影或展示二维码
2. 学生用手机扫描，跳转至签到确认页（或由 App 处理）

### 方式二：粘贴令牌手动签到

1. 以学生账号（如 `noor`）登录，进入 **"考勤管理"**
2. 页面顶部显示 **"令牌手动签到"** 卡片
3. 将讲师分享的令牌粘贴到输入框
4. 点击 **"签到"** 按钮
5. 成功后提示"签到成功"，记录即时写入数据库

### 查看个人出勤记录

签到后页面下方显示每门课的出勤率卡片：
- 课程代码与名称
- 出勤率进度条（绿色 ≥80%、黄色 60–79%、红色 <60%）
- 已出席次数 / 总会话次数
- 低于 80% 时显示预警提示

## 六、常见问题

**Q：令牌过期后还能签到吗？**
A：不能。令牌有效期为 30 分钟，过期后二维码和令牌均失效。需由讲师重新开启新会话。

**Q：学生重复提交令牌会怎样？**
A：系统自动去重（upsert），同一会话内同一学生的签到记录只保留一条，时间会更新为最后一次提交。

**Q：讲师提前结束会话后学生还能签到吗？**
A：不能。后端在 check-in 时会检查 `endedAt`，已关闭的会话拒绝签到并返回错误提示。

**Q：管理员看不到某课程的数据？**
A：若该课程从未开启过签到会话，报告中的"总会话数"为 0，表格为空。需讲师至少开启一次会话并产生记录后才会显示。

**Q：讲师登录后显示"未找到课程信息"？**
A：请确认已运行 `pnpm prisma db seed` 完成数据初始化，且账号角色为 `lecturer`。

## 七、技术说明（供开发参考）

| 项目 | 说明 |
|------|------|
| 签到令牌类型 | JWT（HS256），Payload 包含 `offeringId`、`type: 'attendance'`、`exp` |
| 令牌有效期 | 30 分钟（`Date.now() + 30 * 60 * 1000`） |
| 二维码生成 | 前端 `antd <QRCode>` 组件，value 为完整 JWT 字符串 |
| 实时刷新 | 前端 `setInterval(5000)` 轮询 `GET /api/v1/attendance/live-count/:sessionId` |
| 签到方式 | 后端接受 JWT 令牌或 sessionId 两种形式 |
| 数据模型 | `AttendanceSession` (会话) + `AttendanceRecord` (记录) |
| 防重复 | `@@unique([sessionId, studentId])` 约束 + upsert |

### API 端点汇总

| 方法 | 路径 | 权限 | 功能 |
|------|------|------|------|
| POST | `/api/v1/lms/attendance/sessions` | lecturer | 创建签到会话，返回 JWT token |
| POST | `/api/v1/lms/attendance/check-in` | student | 学生签到（支持 JWT 或 sessionId） |
| PATCH | `/api/v1/lms/attendance/sessions/:id/close` | lecturer | 关闭会话 |
| GET | `/api/v1/lms/attendance/live-count/:sessionId` | lecturer | 获取实时签到人数 |
| GET | `/api/v1/lms/attendance/sessions/offering/:offeringId` | lecturer | 列出课程历史会话 |
| GET | `/api/v1/lms/attendance/records/offering/:offeringId` | admin | 获取课程完整出勤报告 |
| GET | `/api/v1/lms/attendance/records/student/:studentId` | student | 获取学生个人出勤汇总 |
| GET | `/api/v1/lms/attendance/offerings/lecturer/:lecturerId` | lecturer/admin | 获取讲师课程列表 |
