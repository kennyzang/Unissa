### 第 3 步：演示冲突检测（现场解决一个冲突）

**操作步骤**：

1. 选择以下课程（点击卡片添加）：
   - ✅ IFN101 — 周一 09:00–11:00（编程入门，3学分）
   - ✅ IFN102 — 周三 09:00–11:00（数据结构，3学分）
   - ✅ IFN201 — 周二 14:00–16:00（数据库系统，3学分）
   - ✅ ARA101 — 周四 10:00–12:00（阿拉伯语一，3学分）

2. 此时再选择：
   - ⚠️ ARA102 — 周三 09:00–11:00（阿拉伯语二，3学分）

3. **系统立即检测到冲突**：
   - 页面顶部出现红色冲突横幅：「IFN102 与 ARA102 在 Wednesday 09:00–11:00 时间重叠」

**说明词**：「注意看，当我选择 ARA102 后，系统立刻检测到它与 IFN102 在周三上午存在时间冲突，并实时高亮显示。这避免了学生误报名，也避免了后续人工调课的麻烦。现在我移除 ARA102，冲突解决。」

## 系统实现说明（技术参考）

### 后端注册 API（`POST /api/v1/students/:id/register-courses`）

注册时自动执行以下操作：

1. **先修课程验证**：查询学生已完成课程，对比各选课的先修要求
2. **冲突检测**：检查所有选课的 `dayOfWeek + startTime/endTime` 是否存在重叠
3. **学分验证**：检查总学分是否符合学生类型的范围（标准学生：12–18 学分）
4. **创建选课记录**：为每门课程创建 `Enrolment` 记录（`upsert` 确保幂等性）
5. **生成校园卡**：格式为 `CC-{年份}{学号后三位}`（如 CC-2026001）
6. **激活图书馆账号**：创建/更新 `LibraryAccount` 记录，设置 `isActive = true`
7. **激活邮箱标志**：更新 `Student.emailAccountActive = true`
8. **生成费用账单**：自动计算学费（按学分 × 学费单价）+ 图书馆费 - 奖学金减免

### 前端冲突检测（实时）

选课页面在用户选择课程时，实时运行客户端冲突检测：

```typescript
function timesOverlap(a: Offering, b: Offering): boolean {
  return a.dayOfWeek === b.dayOfWeek
    && a.startTime < b.endTime
    && a.endTime > b.startTime
}
```

检测到冲突时：
- 显示红色冲突横幅（说明冲突课程和时间）
- 冲突课程卡片边框变红，显示「⚠ 冲突」徽章
- 注册按钮禁用，直到用户解决冲突
